🔄 エージェントと採用担当者の間でどのようにデータが流れるかを示しています。
1
採用担当者がポータルURLを発行・共有
ATS内部（SCR-13）から AgencyContact を登録すると portal_token が自動生成される。
「URLコピー」ボタンで /agency/portal?token=xxxx をコピーしてエージェントへ共有（メール・Slack等）。
🖥 ATS（採用担当者）
→ 📧 エージェント担当者へ URL 共有
2
エージェントが学生を登録
エージェント担当者がポータル URL を開き「学生を登録する」タブから情報・履歴書を入力。
登録完了後、採用担当者のダッシュボードに通知が届く。
🌐 エージェントポータル
→ 🗄️ Applicant テーブル（agency_contact_id 付き）
3
採用担当者が選考を進める
ATS内部で通常どおり選考を進める（ステージ変更・面接登録等）。
エージェントはポータルの「紹介した学生一覧」タブでリアルタイムに状況を確認できる。
🖥 ATS（内部操作）
↓ エージェントは閲覧のみ
4
採用担当者が合否連絡を送信
SCR-13「合否連絡送信」タブで対象学生・合否・内定通知書テンプレートを選択して送信。
Microsoft Graph API 経由で採用担当者個人アドレスから AgencyContact.email 宛にメールが届く。
内定の場合は内定通知書 PDF が添付される。
📧 Microsoft 365
採用担当者アドレス → エージェントのメール
