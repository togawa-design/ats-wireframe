Webhookエンドポイント
メソッドパス説明
POST /api/webhooks/line LINE Platformからのイベント受信口。署名検証（X-Line-Signature）必須
署名検証: HMAC-SHA256(channel_secret, request_body) を Base64 エンコードした値と X-Line-Signature ヘッダーを比較。不一致は 400 を返す。

認証フロー（Authorization Code Flow）
1
ログインボタン押下
ATS が Google の認可エンドポイントへリダイレクト（state パラメータで CSRF 対策）
2
Google 認証・同意画面
ユーザーが Google アカウントでログイン。組織外アカウントはエラー（hd パラメータでドメイン制限）
3
認可コード受信
ATS の /api/auth/google/callback に code を受信。state を検証
4
トークン交換
Google Token Endpoint に code を送り access_token・refresh_token・id_token を取得
5
Staff レコード作成 or 照合
id_token の sub 値で Staff.google_id を検索。初回ログインは Staff を INSERT（role='recruiter'）。is_active=false はアクセス拒否
6
セッション発行
StaffSession に INSERT。セッションIDを HttpOnly Cookie で発行。AuditLog に login を記録
🔑
トークン管理
トークンTTL保存場所
access_token1時間サーバーサイドのセッションストア（Redis 等）。クライアントには渡さない
refresh_token長期（失効まで）サーバーサイドの暗号化ストレージ。access_token 失効時に自動更新
id_token1時間検証後に破棄。sub・email・name の抽出にのみ使用
セッションID8時間（非操作時は30分で失効）HttpOnly・Secure Cookie。StaffSession で管理

Push Notifications 受信エンドポイント
メソッドパス説明
POST /api/webhooks/google-calendar Google Calendar からのプッシュ通知受信口。ヘッダー X-Goog-Channel-Token で署名検証
🔄
Push Notifications チャンネル管理
1
チャンネル登録
担当者の Google Calendar に対して POST /calendars/{calendarId}/events/watch を呼び出し。有効期限: 7日（最大）
2
通知受信
Webhook エンドポイントで X-Goog-Resource-State ヘッダーを確認。sync / exists / not_exists のいずれかを処理
3
変更内容取得
通知のみでは変更内容が不明なため、GET /calendars/{calendarId}/events/{eventId} で最新データを取得
4
ATS側 DB 更新
google_calendar_event_id で CalendarSlot を特定し、日時・ステータスを更新。google_synced_at を現在時刻に更新
5
チャンネル更新
期限切れ3日前にバッチ処理でチャンネルを再登録（POST /calendars/{calendarId}/events/watch）

エラーハンドリング
ケース処理
署名検証失敗400 を返して処理中断。ログに記録
CalendarSlot が見つからない（ATS側で削除済み）スキップ。Calendar 側イベントはそのまま（孤立イベント）

競合解決ルール
ATS と Google Calendar の両方で同一イベントが変更された場合（二重編集）の解決方針
ケース判定基準処理
ATS編集 vs Google Calendar編集（同時） CalendarSlot.updated_at（ATS） vs Google Calendar イベントの updated 時刻を比較 新しい方を優先して両方に反映。古い変更を上書き
Google Calendar 側でイベント削除 X-Goog-Resource-State: not_exists 受信 is_booked=false の場合のみ ATS 側の CalendarSlot も削除。is_booked=true の場合はスタッフへ警告通知（面接予約が紐づいているため自動削除しない）
Google Calendar 側でイベント日時を過去に変更 Google イベントの start < 現在時刻 ATS の CalendarSlot は更新せずスタッフへ通知。日時の過去変更は手動確認が必要
